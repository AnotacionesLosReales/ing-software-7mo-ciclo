# Semana 4: Agile Project Scope Management

## Intro

- El alcance del proyecto son las caracteristicas y funciones de los entregables que son necesarias para obtener la aprobacion del cliente.
- Gestionar el alcance del proyecto es trabajo del gerente de proyectos; garantiza que todo el trabajo requerido se realice para completar el proyecto con exito.
- La finalizacion del **alcance del producto** se mide frente a los requisitos del producto, mientras que el **alcance del proyecto** se mide frente al plan integrado de gestion del proyecto.
- El enfoque Agile (variable) es distinto del enfoque cascada (fijo) porque el enfoque cambia debido a cambios comerciales o los requisitos del cliente.

## Plan Project Scope Management

- El plan de gestion del alcance del proyecto es el resultado del proceso de planificacion del alcance.
- Este documento define los procesos que se seguiran para definir el alcance, documentar el alcance, verificar y aceptar el alcance y los entregables completados, y gestionar y controlar las solicitudes de cambio del alcance.
- En Agile, el proceso iterativo e incremental es lo que gestiona el alcance.
- A menos que sea para auditoria, no se requiere ningun documento adicional que describa los procedimientos para la gestion del alcance.
- El alcance se define y redefine constantemente en Agile, como parte de las reuniones de planificacion y mediante la gestion del **Product Backlog**.

## Collect Project Requirements

- Es necesario para identificar y documentar lo que necesitan las partes interesadas del proyecto.
- Los requisitos que identificara el cliente y otras partes interesadas deben ser cuantificables, medibles y docmentados para confirmar que los resultados satisfacen a los **stakeholders**.

**Inputs:**
- Project Charter.
- Project Management Plan.
- Documentacion del proyecto.
- Documentacion del negocio.

**Tools and techniques:**
- Brainstorming.
- Entrevistas.
- Focus groups.
- Benchmarking.

**Outputs:**
- Requisitos del negocio.
- Requisitos de stakeholders.
- Requisitos de la solucion.
- Requisitos de calidad.

## Define the Product and Project Scope

- El alcance del **producto** se define como el **Product Backlog**.
- El alcance del **proyecto** contiene el alcance del producto y describe lo que hace y no hace el equipo del proyecto.
